# Host - Backend, Voice UI, and Tone Engine

This directory contains the desktop/web host for the Smart Guitar Amp. One
FastAPI process owns the serial link to the Arduino, keeps the authoritative amp
state, runs the OpenAI-first tone engine, serves the React dashboard, and exposes
JSON/SSE APIs for manual control, chat, voice transcription, and telemetry.

## Layout

```text
app/
  main.py         FastAPI app: /api/* endpoints, static UI serving, app lifetime
  models.py       AmpParams, telemetry, chat schemas, firmware value ranges
  serial_link.py  hardware serial bridge, MockLink fallback, port autodetect
  tone_engine.py  OpenAI/Anthropic tone engine with set_amp_params tool use
  openai_client.py OpenAI client setup for API key, proxy URL, and CA bundle
  devtools.py     Makefile helpers for serial-port detection and config checks
  static/         built React UI generated by make web-build, gitignored
web/              React + Vite frontend source
sf4_serial.py     CLI bench-test tool for sending P frames and reading telemetry
Makefile          setup, build, run, serial, and firmware automation
requirements.txt  pip dependency list, kept in sync with pyproject.toml
pyproject.toml    Python project metadata and dependencies
.env.example      local configuration template; copy to .env
```

The React source lives in `web/`. `make web-build` compiles it into `app/static/`,
which FastAPI serves at `http://127.0.0.1:8000` by default.

## Quick Start

```bash
cd src/host
make env
# edit .env and set OPENAI_API_KEY, OPENAI_BASE_URL if using the Arm proxy,
# OPENAI_CA_BUNDLE if your machine needs the company CA, and SF4_PORT if needed
make install
make run
```

Open `http://127.0.0.1:8000`.

`make run` also installs Python dependencies and builds the UI on first run if the
built frontend is missing. Use `Ctrl+C` to stop Uvicorn.

## Make Targets

| Target | Purpose |
| --- | --- |
| `make help` | Print the available commands. |
| `make env` | Create `.env` from `.env.example` if it does not exist. |
| `make py-install` | Install backend dependencies with `uv sync` when `uv` is available, otherwise create `.venv` and run `pip install -r requirements.txt`. |
| `make web-install` | Install frontend dependencies with `npm ci --include=dev --no-audit --no-fund --progress=false`. |
| `make web-build` | Build the React/Vite UI into `app/static/`. |
| `make install` | Run backend install, frontend install, and frontend build. |
| `make run` | Start FastAPI on `HOST:PORT`, building the UI first if missing. |
| `make dev` | Same as `make run`, but with Uvicorn `--reload`. |
| `make web-dev` | Start Vite hot reload on port 5173; `/api` is proxied to FastAPI on port 8000. |
| `make port` | List detected serial ports and mark likely Arduino ports with `*`. |
| `make set-port` | Write the first likely Arduino port into `.env` as `SF4_PORT=...`. |
| `make serial-perms` | Temporarily run `sudo chmod a+rw $(SF4_PORT)` if Linux denies access to the board. |
| `make doctor` | Print Python, Node, npm, host URL, serial setting, and whether OpenAI settings are present. |
| `make cli ARGS="--demo overdrive"` | Test the serial link without the browser. |
| `make flash` | Compile and upload `../DSP_Pipeline` with `arduino-cli`. |
| `make clean` | Remove `.venv`, Python caches, and built UI output. |

For frontend-only development, use two terminals:

```bash
make run        # terminal A: FastAPI on :8000
make web-dev    # terminal B: Vite on :5173 with /api proxying
```

## Dependencies

System tools:

- Python 3.11 or newer.
- Node.js and npm for the React/Vite frontend.
- `arduino-cli` only if you want `make flash` to compile and upload firmware.
- Linux serial access to the board, either through group membership or `make serial-perms`.

Python packages are declared in both `requirements.txt` and `pyproject.toml`:

- `fastapi`, `uvicorn[standard]`: web server and API runtime.
- `pyserial`: USB serial link to the Arduino.
- `openai`, `httpx`: OpenAI chat/STT client, enterprise proxy, and CA-bundle support.
- `anthropic`: optional fallback provider.
- `pydantic`: API and firmware parameter validation.
- `python-dotenv`: `.env` loading.
- `python-multipart`: browser audio upload support for `/api/transcribe`.

Frontend packages are managed by `web/package.json` and `web/package-lock.json`.
Use `make web-install` or `make install`; do not use `npm install --omit=dev`, because
Vite is a dev dependency required for building.

## Configuration

Copy `.env.example` to `.env` with `make env`. `.env` is gitignored and should hold
machine-specific settings and API keys.

| Variable | Default | Meaning |
| --- | --- | --- |
| `SF4_LLM_PROVIDER` | `openai` | Tone engine provider: `openai` or `anthropic`. |
| `OPENAI_API_KEY` | blank | Required for OpenAI chat and Whisper transcription. |
| `OPENAI_CHAT_MODEL` | `gpt-5.5` | Chat model used for tone generation. Change this if the proxy has no capacity for that model. |
| `OPENAI_BASE_URL` | OpenAI default | Set to the enterprise proxy URL, for example `https://openai-api-proxy.geo.arm.com/api/providers/openai/v1`. |
| `OPENAI_CA_BUNDLE` | Arm root CA path | Optional CA bundle path for enterprise TLS verification. |
| `OPENAI_TRANSCRIBE_MODEL` | `whisper-1` | Speech-to-text model used by `/api/transcribe`. |
| `ANTHROPIC_API_KEY` | blank | Optional fallback when `SF4_LLM_PROVIDER=anthropic`. |
| `SF4_MODEL` | `claude-haiku-4-5` | Anthropic model name for the fallback provider. |
| `SF4_PORT` | auto-detect | Serial device, for example `/dev/ttyACM0`. Use `make port` or `make set-port`. |
| `HOST` / `PORT` | `127.0.0.1` / `8000` | Uvicorn bind address used by `make run` and `make dev`. |

For the Arm enterprise OpenAI proxy, the relevant `.env` entries are usually:

```bash
OPENAI_API_KEY=your_key_here
OPENAI_BASE_URL=https://openai-api-proxy.geo.arm.com/api/providers/openai/v1
OPENAI_CA_BUNDLE=/usr/local/share/ca-certificates/ARM-Enterprise-PKI-Root-CA.crt
OPENAI_CHAT_MODEL=gpt-5.5
OPENAI_TRANSCRIBE_MODEL=whisper-1
```

If you see `data_residency_mismatch` or `No available capacity`, the proxy rejected the
selected model in that region. Change `OPENAI_CHAT_MODEL` to a model that your proxy
supports and restart `make run`.

## API

| Endpoint | Purpose |
| --- | --- |
| `GET /api/state` | Current amp parameters, serialized with the effect name. |
| `POST /api/params` | Apply a full manual `AmpParams` update. |
| `POST /api/chat` | Run one tone-engine turn; may call `set_amp_params` and update the board. |
| `POST /api/transcribe` | Upload browser audio and return Whisper text. |
| `GET /api/telemetry` | Server-Sent Events stream of board telemetry at about 10 Hz. |

The app falls back to `MockLink` when no Arduino is available, so the UI, chat, and
voice flow can be tested without hardware. Manual changes still update the shared app
state, but no serial frames are sent to a board.

## Troubleshooting

- `vite: not found`: run `make web-install` or `make install`. If npm used omitted dev dependencies, run `npm --prefix web ci --include=dev --no-audit --no-fund --progress=false`.
- Npm hangs on VPN: check whether company VPN/proxy rules block `registry.npmjs.org`; disconnecting VPN or configuring npm proxy may be required.
- `Permission denied: /dev/ttyACM0`: run `make set-port`, then `make serial-perms`, or add your user to the Linux dialout group and log out/in.
- `Chat is unavailable`: set `OPENAI_API_KEY` or configure the Anthropic fallback, then restart the server.
- `Transcription error`: check `OPENAI_API_KEY`, `OPENAI_BASE_URL`, `OPENAI_CA_BUNDLE`, and VPN/proxy connectivity.
- Browser says microphone denied: allow microphone access for `http://127.0.0.1:8000` in the browser site settings, then reload.
